* ¿Qué es Shadowing?

Shadowing es una técnica de aprendizaje de idiomas donde escuchas una grabación y repites en voz alta al mismo tiempo o inmediatamente después, imitando EXACTAMENTE la pronunciación, ritmo y entonación. Es como ser la "sombra" vocal del hablante.

Cómo hacerlo en la práctica:

  1. Pon un video de Pacho8a o Rachel's English
  2. Escucha UNA oración completa
  3. Pausa el video
  4. Repite la oración en voz alta, intentando sonar IGUAL (ritmo, tono, velocidad)
  5. Si no te sale, repite 2-3 veces
  6. Avanza a la siguiente oración

Tu escucha ya está en B1/B2, así que esto te va a salir relativamente bien. El objetivo es mejorar tu PRODUCCIÓN oral para que iguale tu comprensión.

* ¿Qué es la terminal?

La terminal (también llamada consola, línea de comandos, o shell) es una ventana donde escribes comandos de texto para controlar tu computadora. En vez de hacer clic en carpetas, escribes instrucciones.

Es como enviar mensajes de texto a tu computadora y que ella responda.

¿Cómo abrirla?

  • Windows: Buscar "PowerShell" o "Terminal" en el menú inicio. O dentro de VS Code: Ctrl+`
  • Mac: Buscar "Terminal" en Spotlight (Cmd+Espacio)
  • Dentro de VS Code: Ctrl+` (la tecla junto al 1) — esta es la que más usarás

* Instalar Git (si no lo tienes)

Primero verifica si ya lo tienes: abre la terminal y escribe:

git --version

Si aparece un número (ej: "git version 2.43.0"), ya lo tienes. Si dice "command not found":

  • Windows: Ve a git-scm.com/download/win, descarga e instala (siguiente, siguiente, finalizar)
  • Mac: Escribe git en terminal, macOS te pedirá instalar las herramientas de developer. Acepta.

Después de instalar, configura tu nombre y email:

git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

* Instalar nvm (Node Version Manager)

nvm te permite instalar y cambiar entre versiones de Node.js. No lo necesitas si ya tienes Node instalado, pero es útil a futuro.

Windows:

  1. Ve a github.com/coreybutler/nvm-windows/releases
  2. Descarga nvm-setup.exe (el más reciente)
  3. Instálalo (siguiente, siguiente, finalizar)
  4. Cierra y reabre la terminal
  5. Escribe: nvm install lts (instala la última versión estable)
  6. Escribe: nvm use lts
  7. Verifica: node --version

Mac/Linux:

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

Cierra y reabre la terminal, luego: nvm install --lts y nvm use --lts.

* Crear SSH Key para GitHub

La SSH key permite que tu computadora se conecte a GitHub sin pedir contraseña cada vez. El curso de Git+GitHub de DevTalles te enseña esto, pero aquí tienes los pasos:

  1. Abre terminal y escribe:
    ssh-keygen -t ed25519 -C "tu@email.com"
  2. Presiona Enter 3 veces (acepta todo por defecto)
  3. Copia tu clave pública:
    Windows: type %USERPROFILE%\.ssh\id_ed25519.pub
    Mac/Linux: cat ~/.ssh/id_ed25519.pub
  4. Ve a github.com/settings/keys
  5. Clic en "New SSH key", pega tu clave, y guarda

* Clonar un repositorio

Clonar = descargar una copia completa de un proyecto de GitHub a tu computadora.

  1. Ve al repositorio en GitHub (ej: github.com/usuario/proyecto)
  2. Clic en el botón verde "Code"
  3. Copia la URL (HTTPS o SSH)
  4. En tu terminal, navega a donde quieras guardarlo:
    cd ~/proyectos
  5. Clona:
    git clone https://github.com/usuario/proyecto.git
  6. Entra a la carpeta: cd proyecto

Nota: esto lo aprenderás en detalle en la Sección 6 del curso de Git+GitHub de DevTalles. No te preocupes si no entiendes todo ahora.

* Crear estructura de carpetas desde terminal

Ejemplo: crear ~/proyectos/semana-1/dia-4/

cd ~
mkdir -p proyectos/semana-1/dia-4
cd proyectos/semana-1/dia-4
echo "Dia 4 completado" > notas.txt
cat notas.txt

mkdir -p crea todas las carpetas necesarias aunque no existan. echo "texto" > archivo crea un archivo con ese texto. cat muestra el contenido.

* Deploy en GitHub Pages

GitHub Pages convierte un repositorio en una página web gratis. Aprenderás esto en el curso de Git+GitHub de DevTalles (Sección 7), pero los pasos básicos son:

  1. Tu repo debe tener un archivo index.html
  2. Ve a Settings → Pages en tu repo de GitHub
  3. En "Source" selecciona "Deploy from a branch"
  4. Selecciona la rama "main" y carpeta "/ (root)"
  5. Clic en Save
  6. En ~2 minutos tu página estará en: https://tuusuario.github.io/tunombrerepo/
